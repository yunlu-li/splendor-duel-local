# 本地试玩说明

## 启动前端

```bash
npm run dev
```

浏览器打开 Vite 输出的本地地址，通常是：

```text
http://localhost:5173/
```

如果本机 `localhost` 解析或端口占用有冲突，可以使用：

```text
http://127.0.0.1:5173/
```

## 启动完整本地服务

好友房需要前端和服务端同时运行：

```bash
npm run dev:full
```

该命令会启动 Vite 前端和 Fastify + Socket.IO 房间服务。

## 当前可玩内容

- 首页可选择本地双人、机器人练习或邀请好友对战。
- 输入/修改 seed 后重开，可复现同一开局。
- 点击棋盘 token 选择，最多 3 个。
- 操作面板支持：
  - `拿已选 token`：执行强制拿 token。
  - `花特权拿 1 个`：选择 1 个非黄金 token 后执行可选特权行动。
  - `补充棋盘`：有空位且袋中有 token 时补盘。
  - `清空选择`：清理当前棋盘选择。
- 市场卡支持：
  - `买`：打开支付面板，选择具体支付 token。
  - `留`：保留明牌并拿棋盘上的黄金。
  - `盲保`：保留对应等级牌堆顶牌。
- 玩家面板支持：
  - 显示 token、特权、威望、皇冠、bonus、同色分。
  - 自己的保留区展示具体卡牌、购买入口和资源缺口。
  - 对手保留区只显示保留数量，不展示具体卡牌。
- 右侧栏支持：
  - 点击已购卡或自己的保留卡后展示完整卡牌详情。
  - 展示诏令日志。
- Awaiting choice 流程：
  - 弃 token。
  - 选择皇室卡。
  - 偷 token。
  - 拿同色 token。
  - Associate 复制 bonus。

## 建议验证路径

1. 进入本地双人模式，选择 1-3 个连续 token，执行拿取。
2. 选择有黄金的开局，保留一张市场卡，确认自己的保留区可见、对手只看到数量。
3. 在资源不足时查看保留卡购买按钮，确认会展示缺口。
4. 获得特权后进入可选阶段，选择 1 个非黄金 token，执行特权拿，确认扣 1 特权并拿到 token。
5. 进入机器人模式，点击执行建议或手动操作，确认机器人能自动行动。
6. 使用 `npm run dev:full` 启动完整服务，创建好友房并用邀请链接打开第二个标签页，双方准备后开始。

## 已知限制

- 好友房暂无断线重连和持久化。
- 机器人策略较简单，适合验证流程，不代表强对弈策略。
- 支付面板已可手动选择，但黄金抵扣提示仍可继续优化。
