# 璀璨宝石对决 网页版

一个可以直接在浏览器里玩的《璀璨宝石：对决》网页原型。适合一个人和机器人练习，也适合同一台电脑上两个人轮流对战。

> 本项目是个人学习和试玩原型，不包含官方素材，也不用于商业用途。

## 直接开玩

打开这个地址即可进入游戏：

```text
https://yunlu-li.github.io/splendor-duel-local/
```

推荐第一次先选 `与机器人对战`，不需要找人，点进去就能开始熟悉流程。

## 可以玩什么

- `与机器人对战`：你操作一方，机器人自动行动，适合一个人试玩。
- `本地双人对战`：同一台电脑、同一个浏览器里两个人轮流操作。
- `邀请好友对战`：需要本地启动完整服务，在线网页试玩版暂不支持。

## 怎么操作

- 点击棋盘上的宝石来选择 token。
- 点击 `拿已选 token` 拿走选中的宝石。
- 有特权时，选中 1 个非黄金 token 后可以点击 `花特权拿 1 个`。
- 点击市场卡下面的 `买` 打开支付面板。
- 点击市场卡下面的 `留` 保留该卡并拿黄金。
- 点击牌堆区域的 `盲保` 保留对应等级的牌堆顶牌。
- 自己的保留区会显示具体卡牌、购买按钮和资源缺口。
- 对手的保留区只显示保留数量，不显示具体卡牌。
- 点击已购买卡或自己的保留卡，会在右侧显示完整卡牌详情。

## 游戏目标

按照《璀璨宝石：对决》的常见胜利条件推进：

- 总威望达到胜利要求。
- 单色威望达到胜利要求。
- 收集足够皇冠并获得皇室卡。

页面顶部会显示当前回合、阶段和双方分数信息。

## 网页版说明

在线试玩版是静态网页，不需要安装任何东西，也不需要注册账号。它可以玩机器人模式和本地双人模式。

好友房需要一个持续运行的房间服务器，所以不能只靠静态网页完成。如果你只是想让朋友快速体验，建议先让对方打开网页玩机器人模式。

## 常见问题

### 打不开网页

先确认地址是：

```text
https://yunlu-li.github.io/splendor-duel-local/
```

如果页面刚发布或刚更新，GitHub Pages 可能需要等 1-2 分钟刷新。

### 点“邀请好友对战”不能玩

这是正常的。在线网页试玩版没有房间服务器，所以好友房入口会被禁用。想体验好友房，需要按下面“本地完整版本”启动。

### 不知道第一步该做什么

选 `与机器人对战`，然后从棋盘拿宝石。资源够了以后，点击市场卡下面的 `买` 购买卡牌。

## 本地完整版本

如果你想在自己的电脑上运行完整版本，包括好友房，需要安装 Node.js 20+。

会用 Git：

```bash
git clone https://github.com/yunlu-li/splendor-duel-local.git
cd splendor-duel-local
npm install
npm run dev:full
```

不会用 Git：

1. 打开项目页面：`https://github.com/yunlu-li/splendor-duel-local`
2. 点击绿色 `Code` 按钮。
3. 点击 `Download ZIP`。
4. 解压 ZIP。
5. 用终端进入解压后的文件夹。
6. 执行：

```bash
npm install
npm run dev:full
```

然后打开：

```text
http://localhost:5173/
```

运行 `npm run dev:full` 的终端窗口要一直开着，关掉终端游戏也会停止。

## 好友房本地玩法

1. 执行 `npm run dev:full`。
2. 打开 `http://localhost:5173/`。
3. 选择 `邀请好友对战`。
4. 创建房间后复制邀请链接。
5. 用另一个浏览器窗口、无痕窗口，或同一局域网内的另一台设备打开邀请链接。
6. 两边都点准备，游戏开始。

如果另一台设备打不开邀请链接，通常是因为不在同一个 Wi-Fi、电脑防火墙拦截了 Node.js，或者运行服务的终端已经关闭。

## 开发者信息

技术栈：

- React
- TypeScript
- Vite
- Fastify
- Socket.IO
- Vitest

常用命令：

```bash
npm test
npm run build
npm run dev
npm run dev:full
```

项目结构：

```text
src/
  client/   # 前端页面、组件和样式
  server/   # 好友房服务端
  shared/   # 规则、类型、卡牌和棋盘数据
  tests/    # 单元测试
docs/       # 规则调研和开发记录
scripts/    # 本地开发辅助脚本
```
